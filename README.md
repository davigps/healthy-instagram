# Healthy Instagram

A Safari Web Extension for iOS (and macOS) that hides distracting UI elements on [instagram.com](https://www.instagram.com). It trims Instagram down to **Stories**, **Home**, **Messages**, and **Profile** — hiding Search, Create, Reels, Notifications, and the home feed.

## How it works

Instagram is a single-page app that re-renders its navigation as you browse. This extension combines:

1. **CSS** (`extension/src/styles.css`) — hides distracting nav items and home feed posts immediately when the page loads
2. **Content script** (`extension/src/content.js`) — finds nav elements by stable attributes (`href`, `aria-label`) and re-applies hiding after DOM changes via a debounced `MutationObserver`
3. **Route watcher** — intercepts `pushState` / `replaceState` / `popstate` so feed hiding and page blocking stay in sync as you navigate
4. **Page blocking** — redirects `/explore/`, `/create/`, and `/accounts/activity/` back to home if reached directly

Selectors are intentionally based on URLs and accessibility labels instead of Instagram's generated CSS class names, which change frequently.

## Project structure

```text
extension/
  manifest.json          # Manifest V3 (Safari 15.4+ / iOS 15+)
  src/
    content.js           # DOM hiding logic, route watcher, page blocking
    styles.css           # CSS hide rules
  icons/                 # Extension icons

ios-template/            # Reference Swift files (generated by converter)
scripts/
  convert-for-safari.sh  # Wraps extension in an Xcode project (macOS only)
.github/workflows/
  build-extension.yml    # Packages Chrome and iOS build artifacts
```

## Install on iPhone (Safari)

Safari Web Extensions on iOS must be distributed inside a native app. You need a Mac with Xcode to build and sideload.

### Option A: Download a pre-built Xcode project (no Mac required to start)

If you do not have the repo cloned locally, use the GitHub Actions workflow to generate installable artifacts:

1. Open **Actions → Build Extension → Run workflow** on this repository
2. When the run finishes, download the **healthy-instagram-ios** artifact
3. On a Mac, unzip and open `HealthyInstagram/HealthyInstagram.xcodeproj` in Xcode
4. Select your iPhone as the run destination, set your signing team, then build and run (⌘R)
5. On iPhone: **Settings → Safari → Extensions** → enable **Healthy Instagram**

> iPhone installation still requires a Mac with Xcode and an Apple Developer account for device signing. CI packages the project; sideloading happens on your Mac.

### Option B: Generate the Xcode project locally

On macOS:

```bash
chmod +x scripts/convert-for-safari.sh
./scripts/convert-for-safari.sh
```

This uses Apple's `safari-web-extension-converter` to create `ios/HealthyInstagram/HealthyInstagram.xcodeproj` with iOS and macOS targets.

Then:

1. Open `ios/HealthyInstagram/HealthyInstagram.xcodeproj` in Xcode
2. Select your iPhone as the run destination
3. Set your Apple Developer Team in Signing & Capabilities for both targets
4. Build and run (⌘R) — this installs the container app on your device
5. On iPhone: **Settings → Safari → Extensions** → enable **Healthy Instagram**
6. Open [instagram.com](https://www.instagram.com) in Safari — Search, Create, Reels, Notifications, and the home feed should be hidden; Stories and Home/Messages/Profile nav remain

### Test on macOS Safari (faster iteration)

1. Open the Xcode project
2. Select the **macOS** scheme and run
3. In Safari: **Safari → Settings → Extensions** → enable Healthy Instagram
4. Visit instagram.com

## Develop without Xcode (CSS/JS only)

You can test the hiding logic in any Chromium browser for quick iteration:

1. Open `chrome://extensions`
2. Enable **Developer mode**
3. Click **Load unpacked** and select the `extension/` folder
4. Visit instagram.com

Or download the **healthy-instagram-chrome** artifact from a **Build Extension** workflow run, unzip it, and load the unzipped folder the same way.

> Note: Chrome loading is for development only. Production target is Safari on iOS.

## What gets hidden

| Element | Selector strategy | Pages |
|---------|-------------------|-------|
| Search / Explore nav icon | `a[href="/explore/"]`, `aria-label="Search"`, `input[placeholder="Search"]`, parent nav rows | All instagram.com pages |
| Create / New post button | `a[href="/create/"]`, `aria-label="New post"`, `svg[aria-label="New post"]`, parent nav rows | All instagram.com pages |
| Reels nav icon | `a[href="/reels/"]`, `aria-label="Reels"`, parent nav rows | All instagram.com pages |
| Notifications nav icon | `a[href="/accounts/activity/"]`, `aria-label="Notifications"`, `svg[aria-label="Notifications"]` | All instagram.com pages |
| Home feed posts | `main article`, feed loading indicators | Home (`/`) only |
| Explore page | Redirect from `/explore/` to home | `/explore/` |
| Create page | Redirect from `/create/` to home | `/create/` |
| Notifications page | Redirect from `/accounts/activity/` to home | `/accounts/activity/` |

## Roadmap

Future UI elements to hide (suggested posts on profile, Reels in DMs, etc.) can be added as new rules in `content.js` and `styles.css`.
